# Deploy — Macro & AI Monitor

A standalone Flask app bundling two read-only research dashboards behind one
discreet shared login:

- `/aibubble` — **AI Bubble Monitor** (twin thermometer: market layer + frontier signals)
- `/econ` — **US Economic Monitor** (FRED indicators, with week / month / quarter / year change)

The landing page (`/`) is a low-key bilingual portal (EN / 中文); each dashboard
also has its own internal language toggle.

---

## 1. Push to a new GitHub repo

```bash
cd macro-ai-monitor
git init && git add -A && git commit -m "Initial commit — Macro & AI Monitor"
gh repo create macro-ai-monitor --private --source=. --push   # or create on github.com and push
```

## 2. Create the Render service (Blueprint)

1. Render dashboard → **New + → Blueprint** → pick this repo. `render.yaml` is detected.
2. Set the environment variables (the Blueprint marks which are `sync: false`):

| Variable | What to set |
|---|---|
| `APP_USERNAME` | A **neutral** name, e.g. `analyst` (avoid the company name) |
| `APP_PASSWORD` | Your own strong passphrase (this + the username are the only login) |
| `FRED_API_KEY` | Free key from <https://fredaccount.stlouisfed.org/apikeys> — drives the weekly econ refresh |
| `ANTHROPIC_API_KEY` | *(optional)* AI commentary / briefs; rules-based fallback if unset |
| `AIBUBBLE_REFRESH_PASSWORD` | *(optional)* password for the AI Bubble refresh button (default `aibubble2026`) |
| `SECRET_KEY`, `REFRESH_TOKEN` | auto-generated by Render |

`SECRET_KEY` and `REFRESH_TOKEN` are generated automatically.

## 3. (Recommended) Weekly wake + refresh

The free plan sleeps after 15 min idle and its disk is **ephemeral** — on a cold
start the econ data reverts to the committed seed snapshot. To keep it fresh:

- Create a free **cron-job.org** job that GETs
  `https://<your-service>.onrender.com/cron/refresh?token=<REFRESH_TOKEN>`
  weekly (e.g. Wed 07:10 US Pacific). This both wakes the service and refreshes
  both dashboards. (The in-app APScheduler also runs Wed 07:00, but only fires
  if the instance happens to be awake — the pinger is the reliable trigger.)
- For always-fresh data without a pinger, upgrade the service to **starter**
  (stays warm) and/or attach a **persistent disk** mounted at the data dir.

## Local run

```bash
cd macro-ai-monitor
pip install -r requirements.txt
SECRET_KEY=local APP_USERNAME=analyst APP_PASSWORD=preview123 PORT=5267 python3 app.py
# open http://localhost:5267  (login: analyst / preview123)
```

## Data refresh model

- **AI Bubble** — mostly keyless (yfinance + free public APIs: vast.ai, npm, Hacker
  News, Google News). The Claude executive brief uses `ANTHROPIC_API_KEY` if set,
  else a rules-based fallback. Seed: `data/aibubble/snapshot.json`.
- **US Econ** — needs `FRED_API_KEY` for the weekly refresh. Seed:
  `data/econ/snapshots/latest.json` (committed). Multi-horizon change values are
  computed from each series' own history, so they work even from the seed.

## Rebuilding the econ UI (only if you change the React frontend)

The econ dashboard under `econ_ui/` is a pre-built React (Vite) bundle. Its source
lives in `../us-econ-monitor-industry/frontend`. To change it:

```bash
cd ../us-econ-monitor-industry/frontend
npm install && npm run build -- --base=/econ/
rm -rf ../../macro-ai-monitor/econ_ui && cp -r dist ../../macro-ai-monitor/econ_ui
```
