      METHODOLOGY & ASSUMPTIONS

      五層架構 — L1 用量地圖(五類 token 消耗與各自經濟模式)與 L2 指標字典是人工策展知識庫(寫死於 KB,非即時);L3 是量化層;L4 用量判讀·矽需求視角與 L5 情境/證偽由 Claude 於刷新時合成(無 API key 走規則後備,頁首 engine 欄標注實際引擎)。

      L3a token 台帳 — 有日期的公開揭露(Google I/O 與法說 9.7T→480T→980T/月、Microsoft FY25Q3「單季逾 100T」、OpenAI DevDay「API 每分鐘約 60 億」≈259T/月、OpenRouter 公開看板)加上 2026 世界策展近似值(標 T3/EST)。年化增速 = (v₂/v₁)^(12/月數)−1,只在同平台內計算(口徑不一,絕不跨平台相比);整體讀值取各平台增速「中位數」以避免被單一大平台主導。

      L3b 價格通縮 — 旗艦模型 API 牌價(官方價格頁,T1;2026 世界點為 T3 估計),以輸入:輸出 = 3:1 混合成 $/M tokens:GPT-4 $37.5 → GPT-5.2 級 $4.31、Claude 3 Opus $30 → Claude 5 級 $8、OpenRouter 市場中位 $0.90 → $0.55。逐族年化變化取「首點→末點」,三族平均 −45.0%/yr。「能力調整後」通縮天生被低估(同價更聰明 ≠ 同 token)。live 刷新時 OpenRouter 免 key API 會更新市場中位點。

      L3c 變現率 — 已實現 $/M = 營收 run-rate ÷ 年化 token 量(全為推算,標 EST):OpenAI $25B(對齊 /payback KB v2 2026-07-02)÷ 480T/月×12 ≈ $4.34/M;Anthropic $12B(對齊 /payback)÷ 估計 130T/月 ≈ $7.69/M。推論成本地板 $0.60/M 由 /aibubble vast.ai H100 $2.16/hr ÷ 假設 1,000 tok/s(=3.6M tokens/hr)推導,T3 估計。

      L3e 剪刀差與綜合分數(0=支出遠超用量,100=需求真實且複利) — capex 增速 +80.6% YoY 引自 /aibubble(2026-07-02,不重算)。token 美元增速 = (1+單位增速)×(1+價格變化)−1。權重寫死:token 單位 vs capex 30%(比值 0.5×→0、2.5×→100)、已實現 $/M 趨勢 25%(−30%/yr→0、+20%/yr→100)、token 美元 vs capex 30%(比值 0.4×→0、1.3×→100)、價格彈性健康度 15%(美元成長係數 0.7→0、1.8→100)。結論門檻:<40 SPEND-AHEAD-OF-USE、40–70 GROWING-BUT-UNPAID、≥70 REAL-AND-COMPOUNDING。

      定位與姊妹卡 — /aibubble 剪刀差用 npm 下載(開發者採用代理),本卡是更深的 token 級讀值,僅引用其 capex 端;/compute 從 capex 推需求,本卡是其鏡像。正典數字(OpenAI $25B、Anthropic $12B、capex +80.6%、vast.ai $2.16/hr)一律標「對齊」出處,不默默複製。

      假設與限制 — token 揭露口徑不一;訂閱營收攤到 token 是推算;OpenRouter 樣本偏長尾;內部消耗無市場價格;成本地板依賴單一吞吐假設。估計就是估計,不當事實陳述。手動密碼刷新,非排程。AI 合成內容為解讀輔助,非投資建議。

      Five-layer design — the L1 usage map (five token-consumption classes with their economics) and L2 indicator dictionary are a hand-curated knowledge base (static, not live). L3 is the quantitative layer; L4 usage read + silicon view and L5 scenarios/falsification are synthesized by Claude on each refresh (rules fallback without an API key; the engine badge shows which ran).

      L3a token ledger — dated public disclosures (Google I/O + earnings 9.7T→480T→980T/month, Microsoft FY25Q3 "over 100T this quarter", OpenAI DevDay "~6B tokens/min on the API" ≈259T/month, OpenRouter public dashboard) plus curated 2026-world approximations (T3/EST). Annualized growth = (v₂/v₁)^(12/months)−1, computed WITHIN a platform only (scopes differ — never compared across platforms); the headline read is the MEDIAN of per-platform growth rates so no single large platform dominates.

      L3b price deflation — flagship API list prices (official price pages, T1; 2026-world points are T3 EST), blended at 3:1 input:output into $/M tokens: GPT-4 $37.5 → GPT-5.2-class $4.31, Claude 3 Opus $30 → Claude 5-class $8, OpenRouter market median $0.90 → $0.55. Per-family annualized change is first-point→last-point; three-family mean −45.0%/yr. Capability-adjusted deflation is understated by construction (smarter at the same price ≠ the same token). On live refresh the keyless OpenRouter API updates the market-median point.

      L3c monetization — realized $/M = revenue run-rate ÷ annualized tokens (all implied, EST): OpenAI $25B (aligned /payback KB v2 2026-07-02) ÷ 480T/month×12 ≈ $4.34/M; Anthropic $12B (aligned /payback) ÷ est. 130T/month ≈ $7.69/M. The $0.60/M serving-cost floor derives from /aibubble's vast.ai H100 $2.16/hr ÷ an assumed 1,000 tok/s (=3.6M tokens/hr), T3 EST.

      L3e scissors & composite (0 = spend far ahead of use, 100 = usage real & compounding) — capex growth +80.6% YoY is cited from /aibubble (2026-07-02, not re-derived). Token-dollar growth = (1+unit growth)×(1+price change)−1. Hard-coded weights: token units vs capex 30% (ratio 0.5×→0, 2.5×→100), realized $/M trend 25% (−30%/yr→0, +20%/yr→100), token dollars vs capex 30% (ratio 0.4×→0, 1.3×→100), price-elasticity health 15% (dollar-growth factor 0.7→0, 1.8→100). Verdict thresholds: <40 SPEND-AHEAD-OF-USE, 40–70 GROWING-BUT-UNPAID, ≥70 REAL-AND-COMPOUNDING.

      Positioning vs siblings — /aibubble's scissors uses npm downloads (a developer-adoption proxy); this card is the deeper token-level read and only cites its capex side. /compute derives demand from capex; this card is its mirror. Canonical numbers (OpenAI $25B, Anthropic $12B, capex +80.6%, vast.ai $2.16/hr) always carry an alignment note — no silent duplicates.

      Assumptions & limits — token disclosure scopes are inconsistent; spreading subscription revenue over tokens is an implied blend; the OpenRouter sample skews long-tail; internal consumption has no market price; the cost floor rests on one throughput assumption. Estimates are estimates, not asserted truth. Manual password-gated refresh only, no schedule. AI synthesis is an interpretive aid — not investment advice.
